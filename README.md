# 🐾 PixelPet

A customizable 8-bit pixel-art desktop pet for Windows. A little cat lives on your
desktop — it sits on top of your windows, wanders around on its own, naps, and reacts when
you hover, click, or drag it. Settings live in the system tray.

> Status: early development. The default **white cat** works today (idle / walk / sleep /
> react, personality-driven behavior). AI-generated pets from your own photos are on the
> roadmap.

## Download

**[⬇️ Get the latest release](https://github.com/guscatalano/PixelPet/releases/latest)** — Windows 10/11.

- **Installer** — `PixelPet-Setup-x.y.z.exe`. Installs to your user profile, adds a Start-menu
  shortcut, and keeps itself up to date automatically.
- **Portable** — `PixelPet-x.y.z.exe`. No install; just run it.

Once it's running, look for the 🐾 icon in the system tray for settings, show/hide, and updates.

> **Heads up — "Windows protected your PC."** The app isn't code-signed yet, so Microsoft
> SmartScreen may warn you the first time. Click **More info → Run anyway**. (Code signing is
> on the roadmap.)

## Features

- **Transparent, always-on-top pet** with per-pixel interaction — only the cat's pixels
  catch the mouse; empty space stays click-through.
- **Drag it anywhere**, click it for a reaction, hover to greet it.
- **Lives on its own** — a personality-weighted ambient loop makes it wander, nap, and idle.
  Sleepy cats nap more; energetic cats roam more.
- **System tray** controls (show/hide, quit).
- **No web engine dependency** — built on Electron, which bundles its own Chromium, so it
  works even if Edge/WebView2 is removed.

## Roadmap

- **AI pet from photos** — point it at photos of your real pet and generate a matching pixel
  pet (pluggable OpenAI / Anthropic providers).
- Hand-authored multi-pose sprite sheets, more species (dog, …).
- More triggers & reactions (music playing, a program launching, time of day) via an open
  trigger→reaction system.
- Settings window: pet manager, personality sliders, pet size.

## Development

Requires Node.js 20+.

```bash
npm install
npm run dev        # launch the pet in development (hot reload)
npm run typecheck  # type-check main + renderer
npm run dist       # build a Windows installer + portable exe into release/
```

### Project layout

```
src/
  main/            Electron main process
    behavior/      the pet's "brain": trigger→reaction engine, personality, ambient loop
    tray.ts        system tray
  preload/         typed IPC bridge (window.pet)
  renderer/pet/    canvas animation engine (source-agnostic clip player)
  shared/          sprite data, palette, sizing constants, shared types
scripts/           dependency-free PNG/ICO generators (build the app icons)
assets/            generated icons (regenerated by `npm run gen:icons`)
```

The cat is authored as a role-indexed pixel grid (`src/shared/catSprite.ts`): each pixel is
a *role* (outline, fur, eye, …), and a palette maps roles to colors. Recoloring is just a
palette swap — the foundation for AI-generated pets.

## Releases

Pushing a `vX.Y.Z` tag triggers the GitHub Actions pipeline, which builds the Windows
installer + portable exe and publishes them to a GitHub Release. Every push/PR also builds
and uploads artifacts so `main` always has a working build. Installed copies **update
themselves** from these releases (via `electron-updater`).

See **[RELEASING.md](RELEASING.md)** for the step-by-step release runbook.
